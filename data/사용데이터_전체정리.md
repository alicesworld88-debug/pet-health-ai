## 1-3. 도메인 온톨로지 (분석 개념 체계)

본 연구의 분석 대상 개념과 관계를 다음과 같이 정의한다. 데이터의 각 필드는 이 온톨로지의 클래스·관계에 대응한다.

```
                    ┌─────────────┐
                    │     Dog     │  (반려견)
                    └──────┬──────┘
                           │ hasAttribute
                           ▼
                    ┌──────────────┐
                    │  LifeCycle   │  자견 / 성견 / 노령견
                    └──────┬───────┘
                           │ commonIn
                           ▼
         ┌──────────────┐  belongsTo  ┌──────────────┐
         │   Disease    │────────────▶│  Department  │  내과/외과/피부과/치과/안과
         │   (질병)     │             └──────────────┘
         └──┬───────────┘
            │ hasSymptom
            ▼
         ┌──────────────┐                 ┌──────────────┐
         │   Symptom    │◀──── mentions ──│   Question   │  (보호자 질문, input)
         │   (증상)     │                 └──────┬───────┘
         └──────────────┘                        │ answeredBy
                                                 ▼
                                          ┌──────────────┐
                                          │    Answer     │  (수의사 답변, output)
                                          │  diagnoses →  │──▶ Disease
                                          └──────────────┘
```

**핵심 클래스**: Dog · LifeCycle · Disease · Symptom · Department · Question · Answer

**주요 관계(Property)**:
| 관계 | 의미 | 데이터 대응 |
|------|------|-------------|
| Disease `belongsTo` Department | 질병 → 진료과 소속 | disease ↔ department |
| Disease `commonIn` LifeCycle | 질병 → 호발 생애주기 | disease ↔ lifeCycle |
| Question `mentions` Symptom | 질문 → 언급된 증상 | input ↔ 추출 증상 키워드 |
| Question `relatedTo` Disease | 질문 → 관련 질병 | input ↔ disease |
| Answer `diagnoses` Disease | 답변 → 진단 질병 | output ↔ disease |

**활용**: 구어체 정규화(VeNom)는 Question이 `mentions`하는 Symptom의 표현을 표준 용어로 매핑한다(예: "절뚝거려요"→파행). 즉 온톨로지의 Symptom 클래스가 **연구질문 2(TF-IDF vs BERT)의 '의미적 유사도'가 측정하려는 개념적 기준**을 제공한다. TF-IDF의 한계는 이 온톨로지 관계(동의어·상위어·질병-증상)를 포착하지 못하는 데서 발생한다.

## 2. 전처리

- **형태소 분석**: KoNLPy Okt (명사/동사/형용사, 불용어 제거)
- **구어체 정규화** (input_clean → input_normalized):
  - 절뚝거려요 → **파행(lameness)** · 실제 **114건** 적용
  - 강아지 → 반려견 · **14,186건**
  - 토해요/구역질 → 구토 · 긁어요/가려워 → 소양감 등
- **3단계 파이프라인**: corpus_raw → corpus_validated → corpus_preprocessed
- **VeNom 정규화 사전**: 대표 6종 + 총 34종, 사전에 없는 표현은 BERT 의미 임베딩이 보완

---

## 3. 평가 설계

- **Ground Truth 50건** (수동 구축): 자견 17 · 성견 17 · 노령견 16
- **Soft Match 채점**: 같은 disease + lifeCycle 이면 정답 (정확한 문서 재현 아님)
- **지표**: Hit@1/3/5, MAP@5, McNemar test, paired bootstrap, ANOVA, Kruskal-Wallis

## 4. 검색 성능 결과

### 4-1. 수동 GT 50건
| 지표 | TF-IDF | Sentence-BERT | 차이 |
|------|--------|---------------|------|
| Hit@1 | 18.0% | **24.0%** | +6.0%p |
| Hit@3 | 48.0% | **52.0%** | +4.0%p |
| Hit@5 | 62.0% | 62.0% | ±0 (동률) |
| MAP@5 | 9.74% | **12.87%** | +3.13%p |

- **생애주기별 Hit@5 (GT 50건)**: 자견 BERT **+17.6%p** · 성견 **TF-IDF +5.9%p** · 노령견 ±0

### 4-2. 전체 검증셋 2,399건 (자동)
| 지표 | TF-IDF | BERT |
|------|--------|------|
| Hit@1 | 23.8% | **24.7%** (+0.9%p) |

- **생애주기별 Hit@1**: TF-IDF 자견 25.4 / 성견 24.6 / 노령 21.4 · BERT 자견 23.7 / 성견 **26.4** / 노령 24.0
- **진료과별 Hit@1**: 내과 24.8/26.0 · 외과 23.0/20.7 · 피부과 24.9/26.9 · 치과 23.3/21.7 · **안과 15.7/30.4 (+14.7%p)**

## 5. 통계 검정 (★ 교수 핵심 대응)

| 검정 | 대상 | 결과 |
|------|------|------|
| McNemar | GT 50건 Hit@1 | χ²=**0.27**, **p=0.61** (불일치쌍 BERT 9 / TF-IDF 6) |
| paired bootstrap (2,000회) | GT 50건 | 95% CI **[-10, +20]%p** (0 포함) |
| McNemar | 전체 2,399건 | χ²=**0.54**, **p=0.46** |
| bootstrap | 전체 2,399건 | 95% CI **[-1.3, +3.1]%p** (0 포함) |
| One-way ANOVA | 진료과 × 질문 길이 | F(4, 21599)=**229.48**, p<0.001 |
| Kruskal-Wallis | 생애주기 × 구어체 변화량 | 유의차 없음 |

**결론**: BERT 방향적 우위지만 **통계적 유의성 미확보** → 평가셋 확대 필요

### 성견 우위 검증 (표본효과 입증)
- **구어체 포함 비율**: 자견 71.5% · 성견 71.9% · 노령 71.6% (차이 0.4%p)
- **명사 비율** (Okt, 각 150쿼리): 자견 42.6% · 성견 43.3% · 노령 43.3% (차이 +0.7%p)
- → 생애주기 간 차이 미미 → 성견 TF-IDF 우위는 **n=17 표본 효과**

## 6. RAG 4방향 답변 비교 (지식iN 300건 평균)

| 방향 | 관련성 | 근거성 |
|------|--------|--------|
| A. Gemini 단독 | 0.759 | 0.769 |
| B. TF-IDF 검색 원문 | 0.565 | 0.723 |
| C. BERT 검색 원문 | 0.619 | **0.810 (근거성 1위)** |
| D. RAG (메인) | 0.682 | 0.799 |

- **RAG의 강점**: 관련성·근거성을 **동시에** 높인 유일한 방식 (균형)
- **환각율**: RAG **28%** vs Gemini 단독 **31.6%** (표본 12→150건 일관) — LLM-judge가 RAG를 과소평가하는 지표 한계

## 7. 추론 비용

| | TF-IDF | BERT |
|---|--------|------|
| 속도 | ~13ms/query | **50ms/query** (3.7×) |
| 메모리 | ~11MB | **63MB** (5.7×) |

- BERT 임베딩은 **사전 계산(batch) → S3 저장**, 실시간엔 질문 1건만 인코딩(50ms)

## 8. 커버리지·분포·기타

- **지식iN 커버리지**: **92.6%** (평균 유사도 0.783 · 중앙값 0.794, 8만 건 무작위 샘플 seed=42)
- **intent 분류 정확도**: **84.7%** (학습 데이터 기준 — hold-out 검증은 향후 과제)
- **질병 분포**: '기타' **76.9%** (Long-tail), GT 50건도 기타 74%로 자연 분포 유지
- **공통 실패**: 수동 분류 10건 중 수술·처치 50% / 이물·응급 30% / 희귀질병 20%
- **intent별 키워드**: 증상(구토 6.7·설사 6.3·식욕 4.3%) / 응급("먹었" 31.1·초콜릿 6.4·삼켰 6.2%) / 처치(수술 27.5·약 26.5·중성화 9.6%)
