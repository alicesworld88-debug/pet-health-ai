# COLLOQUIAL_MAP 구축 근거

## 개요

COLLOQUIAL_MAP은 보호자가 실제로 사용하는 구어체 표현을 수의학 표준 용어로 변환하는 매핑 사전입니다.  
온톨로지 관점에서 **동의어 레이어(Synonym Layer)** 를 구현한 것으로,  
TF-IDF 및 BERT 검색 전 쿼리 정규화 단계에서 적용됩니다.

## 구축 방법론 (반자동, Semi-automatic)

```
Step 1. 탐색 (자동)
  코퍼스 21,604건의 input_clean에서 regex 패턴으로 고빈도 구어체 표현 발굴
  → 등장 건수 기준 후보 목록 생성

Step 2. 검증 (수동)
  실제 등장 문장 샘플 추출 → 표현 형태 및 의미 확인

Step 3. 매핑 확정 (수동)
  수의사 Q&A의 output 텍스트에서 대응 의학 용어 확인 → 매핑 확정
```

## 왜 전문 용어를 알 수 있는가?

수의사 Q&A 데이터셋(AI Hub)의 `output` 텍스트에 이미 수의학 표준 용어가 사용되어 있습니다.

| 보호자 구어체 (input) | 수의사 표준 용어 (output) | 확인 방법 |
|---------------------|------------------------|---------|
| "토해요", "구역질" | "구토", "구토 증상" | output 빈도 상위 단어 |
| "절뚝거려요" | "파행", "파행 증상" | output 빈도 상위 단어 |
| "긁어요", "가려워" | "소양감", "소양증" | output 빈도 상위 단어 |
| "헐떡거려요" | "호흡곤란", "빈호흡" | output 빈도 상위 단어 |

즉, **외부 전문 지식 없이 데이터 자체에서 학습**한 매핑입니다.

| itch, scratch | 가려움증 | 100% | ❌ | — |

**핵심 발견**: clarify 필요 증상 5개 (기침·구토·다음증·부종·피부발적) = 챗봇 되묻기 설계 근거  
→ 구조적 안전장치: 잘못된 retrieval로 위험한 답이 나가는 것을 방지  
→ 데이터 기반 대화 분기 설계 (Data-driven dialogue branching from VeNom)

## 온톨로지 확장 방향

```
현재 (동의어 레이어만 구현):
  구어체 ←→ 의학 표준 용어

향후 (완전 도메인 온톨로지):
  구토 ──[증상]──→ 위장관폐색
  구토 ──[증상]──→ 중독
  구토 ──[증상]──→ 췌장염
  파행 ──[관련 부위]──→ 슬개골
  파행 ──[관련 질환]──→ 슬개골 탈구
  슬개골 탈구 ──[호발 생애주기]──→ 자견, 성견
```

## 한계 및 향후 과제

1. **커버리지 부족**: 현재 36개 항목, 21,604건 중 71%가 '강아지' 한 단어에 집중
2. **장거리 패턴 미처리**: "우리 애가 어제부터 밥을 통 안 먹어요" 같은 문장 패턴
3. **문맥 의존 표현**: "피" (토혈 vs 피부) 같은 중의적 표현은 규칙으로 처리 불가 → ML 필요
4. **Word2Vec 자동화**: 코퍼스로 임베딩 학습 후 유사 단어 클러스터링으로 후보 자동 발굴 가능
